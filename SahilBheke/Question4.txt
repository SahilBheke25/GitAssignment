Under what conditions will Git consider a file that has been deleted as being renamed, and what criteria does Git use to determine that a new file is a renamed version of the deleted file?
Ans) File Content Similarity: if the content of the file that was deleted is similar to the new file, Git may consider the new file as the renamed version of the deleted file.
File Deletion and Creation: A files deletion and the creation of a new file with the similar content are considered as a rename.
